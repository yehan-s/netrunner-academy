# Next.js 架构迁移 - 需求说明

## 目标

- 将现有基于 Vite 的 NetRunner 前端迁移到 Next.js（App Router）架构。
- 在不重写 UI 的前提下，尽量保持现有关卡、网络请求与交互行为完全一致。
- 为后续功能开发提供更清晰的页面结构与扩展能力。

## 范围

- 仅涉及前端构建与运行时架构（打包、入口、目录结构）。
- 不改动业务规则本身（关卡判定、请求模拟、Docs 内容等）。
- 环境变量管理继续通过 `.env.local` 完成。

## 功能性需求

- R1：提供基于 Next.js 的开发命令 `npm run dev`，用于本地开发调试。
- R2：提供生产构建命令 `npm run build`，可在本地通过 `npm run start` 进行生产预览。
- R3：现有 `App.tsx` 渲染的 UI 在 Next.js 下行为一致，不新增虚构关卡或按钮。

## 非功能性约束

- 保持 TypeScript 严格模式，避免引入额外全局状态副作用。
- 迁移过程保持仓库结构清晰，旧架构文件要么删除，要么明确标记为废弃。
- 所有改动需要配套 ADR 与任务文档，并在完成后回写状态。
