# 微信剧情 Story 关卡对齐任务

- [x] 更新 ADR：补充 Story 关卡与微信剧情一一对齐的决策（ADR 0010），并在 ADR 0009 中注明被新方案覆盖的部分。
- [x] `constants.ts`：为剧情模式定义独立的 `story_01`~`story_04` 关卡文案和引导步骤，使标题、描述和 `initialUrl` 与微信聊天内容严格一致（如 CDN 缓存、0.01 元会员、员工搜索、订单越权等）。
- [x] `VirtualBrowser`：为 `story_01_login_outage` 等 Story 关卡提供与剧情匹配的页面 UI（例如微信投放落地页 + JS 版本号展示），避免继续使用与剧情无关的旧登录界面。
- [x] `networkEngine`：确保 `story_*` 关卡的初始流量和后端响应与剧情中的线索保持一致（如通过响应头/静态资源路径给出“新 JS 已加载”等可验证信号）。
- [x] `WeChatStoryApp`：推进按钮在到达带 `targetCaseId` 的消息后必须禁用，并给出明确提示文案（例如“请先完成上一个任务”），只有在对应 `story_*` 关卡通关后才能继续推进剧情。
- [x] 测试：为至少一个剧情 Story 关卡编写或补充 Playwright 端到端测试，用真实操作路径验证“从微信剧情 → 打开 Reqable/浏览器 → 完成 Story 关卡 → 返回剧情继续”的流程可用。
